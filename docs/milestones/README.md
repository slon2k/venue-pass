# Milestones

This folder is the source of truth for milestone details.

## Convention

- One file per milestone: `milestone-xx.md`
- Keep open and closed milestones here
- Include in/out scope, capability breakdown, validation checklist, and closure state in each file

## Planning Model

- `docs/roadmap.md` tracks sequencing and status transitions across milestones
- `docs/milestone-plan.md` is a lightweight pointer to the active milestone
- Detailed execution content belongs in this folder
